# Personal Preferences

## Communication
- Be concise and direct. Skip preamble and summaries.
- Don't restate what I said — just do it.
- Only use emojis if I ask for them.

## Code Style
- Keep solutions simple. Don't over-engineer or add features I didn't ask for.
- Prefer editing existing files over creating new ones.
- Don't add docstrings, comments, or type annotations to code you didn't change.
- Only add comments where the logic isn't self-evident.

## Tools & Environment
- macOS is my primary dev environment, but I also work on remote Linux machines.
- Homebrew for package management on macOS.
- NVM for Node version management, pyenv for Python.
- Ghostty is my terminal, Cursor and VS Code for editors.
- Use zsh as default shell.

## Workflow
- Don't commit unless I ask you to.
- Don't push unless I ask you to.
- When making changes, show me what changed at the end — but keep it brief.
